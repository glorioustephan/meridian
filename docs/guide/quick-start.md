# Quick Start

This guide builds a working counter component from scratch and shows the full round-trip: Meridian source, generated output, and usage in a Next.js page.

If you have not yet installed Meridian, follow the [Installation guide](./installation.md) first.

## Step 1: Create the Meridian source file

Create the file `src/components/Counter.tsx`:

```tsx
// src/components/Counter.tsx
'use client';

import { Component, state, effect } from 'meridian';

interface CounterProps {
  initialCount?: number;
  label?: string;
}

export class Counter extends Component<CounterProps> {
  @state count = this.props.initialCount ?? 0;

  get double(): number {
    return this.count * 2;
  }

  increment(): void {
    this.count = this.count + 1;
  }

  decrement(): void {
    this.count = this.count - 1;
  }

  reset(): void {
    this.count = this.props.initialCount ?? 0;
  }

  @effect
  logChange(): void {
    console.log(`Counter "${this.props.label ?? 'default'}" changed to ${this.count}`);
  }

  render() {
    return (
      <div className="counter">
        <p>
          {this.props.label && <strong>{this.props.label}: </strong>}
          {this.count} (×2 = {this.double})
        </p>
        <button onClick={this.decrement}>−</button>
        <button onClick={this.increment}>+</button>
        <button onClick={this.reset}>Reset</button>
      </div>
    );
  }
}
```

Key things to notice:

- `'use client'` is required on the first line of every Meridian component module.
- `@state count` declares reactive state with an initial value of `this.props.initialCount ?? 0`.
- `get double()` is a pure derived value — no `useMemo`, no manual dependency tracking.
- `increment`, `decrement`, and `reset` are plain methods — no `useCallback`.
- `@effect logChange()` runs after every render where `this.count` or `this.props.label` changed. The compiler infers the dependency array.
- `render()` returns the JSX tree. References to `this.count` and `this.double` resolve to local variables in the generated output.

## Step 2: Run the compiler

```bash
pnpm exec meridian build
```

The compiler processes `src/components/Counter.tsx` and writes the generated output to `.meridian/generated/components/Counter.tsx`.

If you see a message like:

```
[meridian] built 1 file in 42ms
```

the compilation succeeded. If there are errors, the compiler will print them with file name, line number, and a diagnostic code. See the [Diagnostics reference](../api/diagnostics.md) for a full list.

## Step 3: Inspect the generated output

<details>
<summary>View .meridian/generated/components/Counter.tsx</summary>

```tsx
// .meridian/generated/components/Counter.tsx
// Generated by Meridian compiler — do not edit manually.
'use client';

import { useState, useEffect } from 'react';
import type React from 'react';

interface CounterProps {
  initialCount?: number;
  label?: string;
}

export function Counter(props: CounterProps) {
  const [count, setCount] = useState(() => props.initialCount ?? 0);

  const double = count * 2;

  function increment() {
    setCount(count + 1);
  }

  function decrement() {
    setCount(count - 1);
  }

  function reset() {
    setCount(props.initialCount ?? 0);
  }

  useEffect(() => {
    console.log(`Counter "${props.label ?? 'default'}" changed to ${count}`);
  }, [count, props.label]);

  return (
    <div className="counter">
      <p>
        {props.label && <strong>{props.label}: </strong>}
        {count} (×2 = {double})
      </p>
      <button onClick={decrement}>−</button>
      <button onClick={increment}>+</button>
      <button onClick={reset}>Reset</button>
    </div>
  );
}
```

</details>

The generated output is plain, readable React TypeScript. Notice:

- `@state count` became `const [count, setCount] = useState(() => props.initialCount ?? 0)`. The initial value is wrapped in a factory function so it is evaluated once.
- `get double()` became `const double = count * 2` — a plain derived expression.
- `increment`, `decrement`, and `reset` became local `function` declarations. No `useCallback` wrappers.
- `@effect logChange()` became a `useEffect` call with an automatically inferred dependency array of `[count, props.label]`.
- `this.props.x` became `props.x` throughout.
- `this.count = newValue` inside methods became `setCount(newValue)`.

## Step 4: Use the component in a Next.js page

Import the generated component in a Server Component page:

```tsx
// app/page.tsx (Server Component)
import { Counter } from '@meridian/components/Counter';

export default function HomePage() {
  return (
    <main>
      <h1>Meridian Counter Demo</h1>
      <Counter initialCount={5} label="Demo counter" />
      <Counter />
    </main>
  );
}
```

The import path `@meridian/components/Counter` resolves to `.meridian/generated/components/Counter` via the path alias configured in `tsconfig.json`. The Server Component page itself is standard Next.js — no Meridian syntax, no `'use client'`. Meridian client components can be freely placed in any Server Component tree.

## Step 5: Start the dev server

```bash
pnpm dev
```

The `predev` script runs `meridian build` before `next dev` starts, so the generated files are always up to date when the bundler begins. For continuous recompilation during development, run `meridian watch` in a separate terminal:

```bash
# Terminal 1
pnpm exec meridian watch

# Terminal 2
pnpm exec next dev
```

## Summary

The full workflow:

1. Write a Meridian class in `src/` with `'use client'`, decorators, and a `render()` method.
2. Run `meridian build` (or `meridian watch` for continuous recompilation).
3. Import the generated output from `.meridian/generated/` (or via the `@meridian/*` alias).
4. Use the component in any Server Component, layout, or page.
