# Debounce Primitive

This example builds a `UseDebounce<T>` Primitive that delays updating a value until a specified number of milliseconds have passed since the last change. It demonstrates the full Primitive authoring pattern: constructor parameter capture, `@state`, `@effect` with cleanup, and `resolve()`.

## The Primitive source

```ts
// src/primitives/UseDebounce.ts
import { Primitive, state, effect } from '@meridian/meridian';

export class UseDebounce<T> extends Primitive<T> {
  @state current: T;

  constructor(private value: T, private delay: number) {
    super();
    this.current = value;
  }

  @effect
  sync(): () => void {
    const timeoutId = setTimeout(() => {
      this.current = this.value;
    }, this.delay);
    return () => clearTimeout(timeoutId);
  }

  resolve(): T {
    return this.current;
  }
}
```

### How it works

- `value` and `delay` are the hook's parameters. They are captured as `private` constructor fields so they can be read as `this.value` and `this.delay` throughout the class body.
- `@state current` holds the debounced value. It is initialized to the first `value` received, so the hook's return value is immediately valid on first render.
- `@effect sync()` starts a timer whenever `value` or `delay` changes. When the timer fires, it updates `this.current` to match `this.value`. The cleanup function cancels the timer if `value` or `delay` changes before it fires — this is the classic debounce pattern.
- `resolve()` returns `this.current`, the debounced value.

The compiler infers the dependency array for `sync()` by scanning the method body and finding reads of `this.value` and `this.delay`. Both are constructor parameters (treated like props in a Component), so the dep array is `[value, delay]`.

import { useState, useEffect } from 'react';

export function useUseDebounce<T>(value: T, delay: number): T {
  const [current, setCurrent] = useState<T>(() => value);

  useEffect(() => {
    const timeoutId = setTimeout(() => {
      setCurrent(value);
    }, delay);
    return () => clearTimeout(timeoutId);

The class becomes a `useUseDebounce` function. The constructor parameters become hook arguments. `resolve()` becomes the return statement. This is exactly the custom hook you would write by hand.

## A component that uses it

```tsx
// src/components/DebouncedInput.tsx
'use client';

import { Component, state, use } from '@meridian/meridian';
import { UseDebounce } from '@meridian/primitives/UseDebounce';

interface DebouncedInputProps {
  onDebouncedChange: (value: string) => void;
  placeholder?: string;
  delayMs?: number;
}

export class DebouncedInput extends Component<DebouncedInputProps> {
  @state value = '';

  @use(UseDebounce, () => [this.value, this.props.delayMs ?? 300])
  debouncedValue!: string;

  handleChange(e: React.ChangeEvent<HTMLInputElement>): void {
    this.value = e.target.value;
  }

  @effect
  notifyParent(): void {
    this.props.onDebouncedChange(this.debouncedValue);
  }

  render() {
    return (
      <div>
        <input
          value={this.value}
          onChange={this.handleChange}
          placeholder={this.props.placeholder}
        />
        {this.value !== this.debouncedValue && (
          <span className="debouncing-indicator">Debouncing...</span>
        )}
      </div>
    );
  }
}
```

### How @use wires in the Primitive

`@use(UseDebounce, () => [this.value, this.props.delayMs ?? 300])` compiles to:

```tsx
const debouncedValue = useUseDebounce(value, props.delayMs ?? 300);
```

The field name `debouncedValue` becomes the local variable. The args factory provides the current value of `value` (the state variable) and `props.delayMs ?? 300` on every render.

The `@effect notifyParent()` reads `this.debouncedValue`. The compiler traces `debouncedValue` → it is a `@use` field, whose underlying hook produces a value treated as a state-like dependency. The dep array becomes `[debouncedValue, props.onDebouncedChange]`.

## Usage in a page

```tsx
// app/page.tsx (Server Component)
import { DebouncedInput } from '@meridian/components/DebouncedInput';

export default function SearchPage() {
  return (
    <main>
      <h1>Debounced Input</h1>
      <DebouncedInput
        onDebouncedChange={query => {
          // This callback fires at most once per 500ms after the user stops typing
          console.log('Search:', query);
        }}
        placeholder="Type to search..."
        delayMs={500}
      />
    </main>
  );
}
```
