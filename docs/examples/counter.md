# Counter

The counter is the canonical Meridian example. It demonstrates `@state`, a getter, a method, and `@effect` in a small, self-contained component.

## Meridian source

```tsx
// src/components/Counter.tsx
'use client';

import { Component, state, effect } from 'meridian';

interface CounterProps {
  initialCount?: number;
  step?: number;
}

export class Counter extends Component<CounterProps> {
  @state count = this.props.initialCount ?? 0;

  get step(): number {
    return this.props.step ?? 1;
  }

  get doubled(): number {
    return this.count * 2;
  }

  increment(): void {
    this.count = this.count + this.step;
  }

  decrement(): void {
    this.count = this.count - this.step;
  }

  reset(): void {
    this.count = this.props.initialCount ?? 0;
  }

  @effect
  logChange(): void {
    console.log(`Count changed to: ${this.count}`);
  }

  render() {
    return (
      <div className="counter">
        <p>
          Count: <strong>{this.count}</strong>
          {' '}(×2 = {this.doubled}, step = {this.step})
        </p>
        <button onClick={this.decrement}>− {this.step}</button>
        <button onClick={this.increment}>+ {this.step}</button>
        <button onClick={this.reset}>Reset</button>
      </div>
    );
  }
}
```

## Walkthrough

Each part of the Meridian source maps directly to a React concept:

| Meridian syntax | What it means | Generated React |
|---|---|---|
| `@state count = this.props.initialCount ?? 0` | Reactive state initialized from a prop | `const [count, setCount] = useState(() => props.initialCount ?? 0)` |
| `get step()` | Pure derived value | `const step = props.step ?? 1` |
| `get doubled()` | Pure derived value | `const doubled = count * 2` |
| `increment()` | Event handler | `function increment() { setCount(count + step); }` |
| `this.count = ...` inside a method | State mutation | `setCount(...)` |
| `@effect logChange()` | Side effect with inferred deps | `useEffect(() => { ... }, [count])` |
| `this.count` in `render()` | Read current state value | `count` |
| `this.step` in `render()` | Read derived value | `step` |

The `@effect logChange()` method reads `this.count` in its body. The compiler scans the method, finds that `this.count` is a `@state` field, and emits `[count]` as the dependency array. The effect runs after the initial render and after every render in which `count` changed.

interface CounterProps {
  initialCount?: number;

export function Counter(props: CounterProps) {
  const [count, setCount] = useState(() => props.initialCount ?? 0);

    setCount(count + step);

## Using the counter in a Next.js page

```tsx
// app/page.tsx (Server Component)
import { Counter } from '@meridian/components/Counter';

export default function HomePage() {
  return (
    <main>
      <h1>Counter Demo</h1>

      {/* Default: starts at 0, increments by 1 */}
      <Counter />

      {/* Custom initial count and step */}
      <Counter initialCount={10} step={5} />
    </main>
  );
}
```

## Key observations

**No `useCallback` on methods.** `increment`, `decrement`, and `reset` are plain function declarations. They do not need `useCallback` because the React Compiler handles memoization when it is beneficial. Meridian does not add defensive `useCallback` wrappers.

**Getter deps are flattened.** `get doubled()` reads `this.count`. If a future effect were to read `this.doubled`, the compiler would trace `doubled → count` and include `count` in the dep array — not `doubled` itself, since `doubled` is not a hook value.

**Initializer runs once.** `@state count = this.props.initialCount ?? 0` captures the initial prop value at mount. If `props.initialCount` changes later, `count` does not reset. This is standard `useState` semantics. To reset on prop change, add an `@effect` that reads the new prop value and updates the state.
