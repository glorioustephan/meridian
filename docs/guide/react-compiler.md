# React Compiler

The [React Compiler](https://react.dev/learn/react-compiler) (formerly "Forget") automatically memoizes components and hooks to eliminate unnecessary re-renders. Meridian-generated output is designed to be valid, clean input for the React Compiler.

## How they fit together

Meridian and the React Compiler operate at different stages:

```
Meridian source (.tsx)
    ↓ meridian build (compile-time transform)
Generated React TSX
    ↓ React Compiler (bundler-time optimization)
Memoized React code
    ↓ next build / vite build
Production bundle
```

Meridian handles authoring ergonomics. The React Compiler handles optimization. Neither depends on the other for correctness.

## What Meridian emits

Meridian's code generation is deliberately minimal:

- **Getters** lower to plain `const` expressions — not `useMemo`. The React Compiler can memoize these when it determines it is beneficial.
- **Methods** lower to plain `function` declarations — not `useCallback`. The React Compiler can memoize these when they are passed to children that have been memoized.
- **Effects** emit dependency arrays that Meridian can prove statically, but no manual memoization is added to values used in those arrays.

This is intentional. Manual `useMemo` and `useCallback` in generated code would conflict with the React Compiler's analysis, add noise to the generated output, and duplicate work. Meridian's correctness does not depend on memoization.

## Compatibility

All valid Meridian-generated output passes the React Compiler's analysis. Meridian does not emit patterns that would cause the React Compiler to bail out of a component, such as:

- Mutation of values outside of event handlers
- Calling hooks conditionally
- Refs read during render in ways that violate rules

The Meridian compiler's own static analysis catches these patterns at build time and rejects them with diagnostic errors. By the time the React Compiler processes a generated file, the file is clean.

## Enabling the React Compiler in Next.js

If you are using Next.js 15 or later, you can enable the React Compiler in `next.config.ts`:

```ts
// next.config.ts
import type { NextConfig } from 'next';

const nextConfig: NextConfig = {
  experimental: {
    reactCompiler: true,
  },
};

export default nextConfig;
```

Install the babel plugin as a dev dependency if you are not already using it:

```bash
pnpm add -D babel-plugin-react-compiler
```

With this configuration, the React Compiler processes all components — including those generated by Meridian — during the Next.js build step.

## No redundant optimization needed

Because Meridian methods lower to plain function declarations, you might wonder whether to wrap them in `useCallback` manually before passing to memoized children. The answer is no.

Before React Compiler:

```tsx
// Generated by Meridian
function Counter(props: {}) {
  const [count, setCount] = useState(0);

  function increment() {
    setCount(count + 1);
  }

  return <button onClick={increment}>{count}</button>;
}
```

With the React Compiler enabled, the compiler determines whether `increment` needs to be stable across renders and applies the appropriate optimization. You do not need to manually annotate anything.

If you are not using the React Compiler, Meridian-generated code is still correct — you may observe that child components re-render more than necessary when passed inline function props, but this is standard React behavior, not a Meridian-specific issue.

## Checking compiler output

To verify that the React Compiler is processing your Meridian-generated files, use the [React DevTools](https://react.dev/learn/react-developer-tools) Profiler. Components optimized by the React Compiler display a "Memo ✓" badge in the component tree.

You can also use the React Compiler playground to paste generated Meridian output and see what the compiler produces.

## Known interactions

**None.** As of v1, there are no known incompatibilities between Meridian-generated output and the React Compiler. If you encounter a case where a generated file causes the React Compiler to bail out, please open an issue in the Meridian repository with the generated file contents. Meridian's code generation rules should be updated to avoid the offending pattern.
